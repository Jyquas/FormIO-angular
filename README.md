# FormIO-angular

## Regula Forensics Web Components integration for AngularJS 1.8.3

This repo contains a drop-in integration under `regula-integration/` that keeps your app on AngularJS 1.8.3 while using `@regulaforensics/vp-frontend-document-components`.

### Why a wrapper bundle (Option B)

`@regulaforensics/vp-frontend-document-components` is ESM-first. Legacy script-tag AngularJS apps cannot consume it directly without a build step. The smallest stable approach is a tiny wrapper bundle that:

- imports and initializes Regula once,
- exposes a global bridge (`window.RegulaWebComponentBridge`) for AngularJS,
- keeps modern package concerns out of the legacy app code.

### Minimal file structure

```text
regula-integration/
  package.json
  esbuild.config.mjs
  src/
    regula-wrapper.js
  legacy/
    regula.module.js
    regula-sdk.service.js
    regula-reader.component.js
    sample/
      index.html
      sample.controller.js
      sample.template.html
  dist/
    regula-wrapper.iife.js   # generated by npm run build
```

### Install and build (npm 9.5.0 lifecycle)

From repository root:

```bash
cd regula-integration
npm install
npm run build
```

`npm run build` now uses npm lifecycle hooks:

- `prebuild` → removes old `dist/` output (`npm run clean`)
- `build` → runs `node esbuild.config.mjs`
- `postbuild` → verifies `dist/regula-wrapper.iife.js` exists

### Run locally

To build and serve the AngularJS sample at `http://localhost:8080`:

```bash
cd regula-integration
npm run run:sample
```

For iterative development (watch + rebuild):

```bash
cd regula-integration
npm run dev
```

This package is pinned to npm 9.5.0 via `packageManager` and `engines` in `regula-integration/package.json`.

### How to wire into an existing AngularJS 1.8.3 app

1. Add `legacy.regula` to your AngularJS module dependencies.
2. Load scripts in this order:
   1. AngularJS 1.8.3
   2. `dist/regula-wrapper.iife.js`
   3. `legacy/regula.module.js`
   4. `legacy/regula-sdk.service.js`
   5. `legacy/regula-reader.component.js`
3. Use the component:

```html
<regula-reader
  is-open="vm.readerOpen"
  license="vm.regulaLicense"
  on-process-finished="vm.handleProcessFinished(result)"
  on-close="vm.handleClose(event)">
</regula-reader>
```

### Initialization details implemented

The wrapper performs the requested Regula setup:

- `window.RegulaDocumentSDK = new DocumentReaderService()`
- `recognizerProcessParam.processParam.scenario = InternalScenarios.MrzAndLocate`
- `recognizerProcessParam.processParam.multipageProcessing = true`
- `imageProcessParam.processParam.scenario = InternalScenarios.MrzAndLocate`
- `defineComponents().then(() => window.RegulaDocumentSDK.initialize())`

### License for non-test environments

Pass a production Base64 license when opening the component (`license="vm.regulaLicense"`), or wire your own source in controller/service config.

A commented example is in `src/regula-wrapper.js`:

```js
// return sdkInstance.initialize({ license: 'YOUR_BASE64_LICENSE_KEY' });
```

### Lifecycle and event behavior

- `PROCESS_FINISHED` and `CLOSE` are bound on the `document-reader` custom element.
- AngularJS digest-safe callbacks use `$scope.$applyAsync(...)`.
- `PROCESS_FINISHED` callback runs only for completed results.
- `CLOSE` forces `isOpen` to `false` and calls `onClose`.
- `$onDestroy` unbinds listeners and calls SDK `shutdown()`.

### Known limitations / assumptions

- The sample assumes one active reader instance. If you need multiple concurrent readers, add reference counting before global `shutdown()`.
- Exact result payload shape can vary by SDK version; completion checks are defensive (`completed`, `status`, and `action`).
- `dist/regula-wrapper.iife.js` is build output and should be generated in CI/CD or during integration.
